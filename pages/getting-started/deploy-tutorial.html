从0到上线：用AI做网站 + 上传GitHub + Vercel部署 完整教程

📅 2026-05-15 ⏱ 20分钟 入门
教程 部署 GitHub Vercel 零基础

这篇教程写给完全零基础的你。你不需要知道什么是GitHub，不需要用过终端，甚至不需要知道"部署"是什么意思。只要你会用浏览器、会打字，跟着这篇教程一步一步来，你就能拥有一个在互联网上真正能访问的网站。

在开始之前

先花一分钟搞清楚我们要做什么。读完这篇教程，你将完成这几件事：

  1. 用AI做一个网站 — 通过和AI对话，让AI帮你生成一个完整的网页
  2. 上传到GitHub — 把你的网站文件存放在一个叫GitHub的地方（可以理解为"网上的文件夹"）
  3. 用Vercel部署上线 — 通过Vercel把你的网站发布到互联网上，得到一个可以分享给任何人的链接

整个过程完全免费，不需要写一行代码，不需要懂任何编程知识。

你需要准备什么？

  • 一台能上网的电脑 — Windows或Mac都可以
  • 一个浏览器 — 推荐用Chrome（谷歌浏览器），Edge或Safari也可以
  • 一个邮箱地址 — 用来注册账号，QQ邮箱、163邮箱、Gmail都行

全部准备好之后，我们开始。

第一步：用AI做出你的网站

在把网站部署上线之前，我们首先需要有一个网站文件。这个文件不用你自己写代码，让AI来帮你写。

1.1 打开AI工具

打开你的浏览器，访问以下任意一个网站：

  • ChatGPT — 在地址栏输入 chat.openai.com 并按回车
  • Claude — 在地址栏输入 claude.ai 并按回车

如果还没注册，用你的邮箱注册一个账号。这两个工具都提供免费套餐，够用了。

什么是浏览器地址栏？ 就是浏览器最上面那条长条框，里面通常显示着"www.baidu.com"或"www.google.com"之类的文字。点击它，输入网址，按回车。

1.2 把这段话复制给AI

进入ChatGPT或Claude的对话界面后，你会看到一个输入框，底部写着"Send a message..."或"输入..."。复制下面这段文字，粘贴进去，按回车发送：

请帮我生成一个个人简历网页的完整HTML代码。要求：
1. 风格简洁现代
2. 包含个人信息、工作经历、项目展示、联系方式
3. 使用深色和浅色两种配色，有一个切换按钮
4. 响应式设计，手机也能看
5. 所有代码放在一个HTML文件中

请直接给我完整的HTML代码，我复制保存就能用。

等几秒钟，AI就会回复一大段代码。你不需要看懂这些代码，只需要做下面一步。

1.3 把代码保存成HTML文件

AI回复之后，你要做的是把这段代码"拿出来"保存成一个文件。具体步骤：

第一步：全选并复制代码

  1. 把鼠标移到AI回复的代码框里
  2. 如果有"Copy"或"复制"按钮，直接点击它
  3. 如果没有复制按钮，用鼠标从代码最开头按住左键拖到最末尾，选中全部代码
  4. 按键盘上的 Ctrl + C（Mac用户按 Command + C）复制
  5. 或者右键点击选中的文字 → 选择"复制"

第二步：打开记事本

  1. Windows用户：点击屏幕左下角的"开始"按钮 → 搜索"记事本" → 点击打开
  2. Mac用户：打开"启动台" → 搜索"文本编辑" → 打开

第三步：粘贴并保存

  1. 在记事本里按 Ctrl + V（Mac按 Command + V），把代码粘贴进去
  2. 点击菜单栏的"文件" → "另存为"（或按 Ctrl + S）
  3. 在弹出的窗口里：
  4. a. 在"文件名"那里输入 index.html
  5. b. 在"保存类型"那里选择"所有文件 (.)"（这一步非常关键！）
  6. c. "编码"选择 UTF-8
  7. d. 选择一个好找的位置，比如"桌面"
  8. e. 点击"保存"

千万注意： 很多人第一次保存的时候，文件名看起来是 index.html，实际上却是 index.html.txt。这是因为Windows默认隐藏了文件后缀名。保存后，如果文件图标是一个浏览器图标（而不是记事本图标），就说明保存对了。如果还是记事本图标，说明后缀名不对，重新保存一次，记得选"所有文件"。

第四步：打开看看效果

在桌面上找到你刚才保存的 index.html 文件，双击它。你的浏览器会自动打开，显示出你的人生第一个网站！

如果你有VSCode（一款免费的代码编辑器），也可以用VSCode来保存文件，操作更简单：打开VSCode → 新建文件 → 粘贴代码 → 按 Ctrl + S → 文件名输入 index.html → 保存。

第二步：创建GitHub账号

现在你的网站还在你自己的电脑上。要让别人也能访问，我们需要把它放到互联网上。第一步是注册一个GitHub账号。

GitHub是什么？ 你可以把它理解成一个"存放在云端的文件夹"，你可以把你的文件放上去，随时查看和管理。全世界很多程序员都用它来存放代码。它完全免费。

2.1 打开GitHub网站

在浏览器地址栏输入 github.com，按回车。

2.2 点击注册

在页面右上角，你会看到一个绿色的"Sign up"按钮，点击它。

2.3 填写注册信息

你会看到一个注册表单，依次填写：

  1. Email（邮箱地址） — 输入你的邮箱，比如 example@qq.com
  2. 点击绿色的"Continue"按钮
  3. Password（密码） — 设置一个密码，至少8位，建议包含字母和数字
  4. 点击"Continue"
  5. Username（用户名） — 给自己取一个名字，比如你的英文名或拼音，如果被占了可以加几个数字
  6. 点击"Continue"
  7. 验证题 — GitHub会出一道简单的拼图题，确认你不是机器人，按提示操作就行
  8. 点击"Create account"

2.4 验证邮箱

注册完成后，GitHub会给你注册的邮箱发一封验证邮件。去你的邮箱找到这封邮件（如果收件箱里没有，看看垃圾箱），点击邮件中的"Verify email address"按钮。

2.5 选择套餐

验证成功后，GitHub会问你选择什么套餐。选择"Free"（免费版），不需要输入任何支付信息。按引导完成设置即可。

现在你已经拥有了一个GitHub账号！

第三步：在GitHub上创建仓库

有了账号之后，你需要创建一个"仓库"来存放你的网站文件。"仓库"可以理解成一个文件夹。

3.1 开始创建仓库

登录GitHub后，你会看到页面上方有一个加号图标"+"。点击它，在弹出的菜单中选择"New repository"。

小提示： 你也可以直接在浏览器地址栏输入 github.com/new，按回车，也会直接跳转到创建仓库页面。

3.2 填写仓库信息

在创建仓库页面，你需要填写以下内容：

  1. Repository name（仓库名称） — 输入 my-website（你也可以自己取一个名字，比如 my-first-site、personal-website，都可以）
  2. Description（描述） — 可以不填，也可以写一句介绍，比如"我的第一个网站"
  3. Public / Private — 一定要选择"Public"（公开）。选Private的话，Vercel部署会有问题。
  4. Initialize this repository with — 不要勾选 "Add a README file"，也不要选 ".gitignore" 或 "license"。保持默认，全部不勾选。

3.3 点击创建

检查无误后，点击页面底部绿色的"Create repository"按钮。

现在你被带到一个新的页面。这个页面上有很多英文字母和一些看起来像代码的命令，别紧张，往下看。

第四步：上传你的网站文件

这一步是最容易让新手懵圈的。很多人看到页面上那些命令就慌了。别怕，你完全不需要那些命令。我们用一个更简单的方法——直接拖拽上传。

4.1 找到上传入口

在你刚创建的仓库页面中，注意看中间区域。有两种方式可以上传文件：

方式一（推荐）： 页面中应该有一行文字"…or create a new repository on the command line"，在这行文字的下方，有一个蓝色的链接文字"uploading an existing file"。点击这个"uploading an existing file"链接。

方式二： 如果没找到方式一，就找页面中间靠右的一个绿色按钮"Add file"。点击它 → 在弹出菜单中选择"Upload files"。

4.2 上传文件

点击上传之后，你会看到一个文件上传页面。现在操作非常简单：

  1. 打开你电脑上存放 index.html 的文件夹（比如桌面）
  2. 用鼠标按住 index.html 文件
  3. 把它拖到浏览器窗口中，松开鼠标
  4. 你会看到文件出现在页面中，显示为一个蓝色标签，写着"index.html"

如果你不想用拖拽，也可以点击页面中的"choose your files"链接，然后在弹出的文件选择窗口中找到 index.html，双击或点击"打开"。

上传之后文件没有出现？ 确认你的文件名是 index.html（不是 index.html.txt）。如果文件图标是记事本图标，说明你之前保存时没选对格式。回到第一步重新保存一次。

4.3 提交文件

文件上传之后，把页面往下滚动一点，你会看到一个"Commit changes"的区域：

  1. 在第一个输入框里，输入提交信息：第一次提交 - 我的网站
  2. 下面有一个"Commit directly to the main branch"的选项，保持它被选中（默认就是选中的）
  3. 点击绿色的"Commit changes"按钮

4.4 检查上传结果

点击提交之后，页面会自动刷新。你会看到你的仓库里多了一个文件——index.html。点击这个文件名，可以查看文件的内容，确认代码正确。

恭喜！你的网站文件已经成功上传到GitHub了。现在它安全地存放在云端，但是别人还不能直接访问它。接下来我们就用Vercel来让它上线。

第五步：注册Vercel

Vercel是一个部署平台，它的作用就是把你在GitHub上的网站文件"发布"到互联网上，让任何人都能通过链接访问。

5.1 打开Vercel网站

在浏览器中打开一个新的标签页，输入 vercel.com，按回车。

5.2 开始注册

在Vercel首页的右上角，点击"Sign Up"按钮。

5.3 选择用GitHub登录

你会看到几个注册选项：

  • "Continue with GitHub" — 选择这个
  • "Continue with GitLab" — 不要选这个
  • "Continue with Bitbucket" — 不要选这个
  • "Continue with Email" — 也可以，但用GitHub最方便

点击"Continue with GitHub"。

5.4 授权Vercel

点击之后，浏览器会跳转到GitHub的授权页面。这个页面在问你是否允许Vercel访问你的GitHub账号。操作很简单：

  1. 如果页面问你要授权哪些仓库，选择"All repositories"（所有仓库）或只选择你刚创建的"my-website"
  2. 点击绿色的"Install"或"Authorize vercel"按钮

5.5 完成注册

授权完成后，你会自动跳转回Vercel网站，并且已经登录成功了。现在你会看到Vercel的管理后台（Dashboard）。

不需要手机验证？ 对，因为你是用GitHub账号登录的，GitHub已经验证过你的身份了，所以Vercel不需要你再注册一次。

第六步：用Vercel部署你的网站

这是最关键的一步。做完这一步，你的网站就正式上线了！

6.1 添加新项目

在Vercel的Dashboard页面中：

  1. 找到"Add New"按钮（通常是深蓝色或灰色的，在页面右上角或中间偏上位置）
  2. 点击它 → 在弹出的菜单中选择"Project"

6.2 导入你的GitHub仓库

接下来Vercel会显示你的GitHub仓库列表：

  1. 在列表中找到你刚才创建的"my-website"（也可能显示为你的用户名/my-website）
  2. 点击仓库右侧的"Import"按钮

如果列表中没有出现"my-website"：

  • 检查一下你是否在创建仓库时选了"Private"（私有），如果选了，需要在Vercel中调整权限
  • 点击"Add GitHub App"或"Configure GitHub App"来添加仓库权限
  • 或者重新创建一个Public的仓库

6.3 配置项目

点击"Import"之后，你会进入"Configure Project"配置页面。这里有很多设置项，但你什么都不用改：

  • Framework Preset（框架预设） — 保持默认的"Other"（不要选Next.js、React等）
  • Root Directory（根目录） — 留空
  • Build Command（构建命令） — 留空
  • Output Directory（输出目录） — 留空
  • Environment Variables（环境变量） — 不需要添加

最容易出错的地方： Framework Preset一定不要选错。如果你选了"Next.js"或"React"，部署会报错，因为我们的网站只是一个简单的HTML文件，不需要任何框架。保持"Other"就对了。

6.4 点击部署

把页面往下滚动，找到底部的一个大大的按钮——写着"Deploy"（部署）。点击它。

6.5 等待部署完成

点击"Deploy"之后，你会进入一个等待页面。Vercel正在自动构建和部署你的网站。你会看到一些日志文字在滚动——不用管它们，等着就行。

这个过程通常需要30秒到1分钟。你可以看到进度条在推进。

6.6 部署成功！

当部署完成时，你会看到：

  • 页面上出现彩色的礼花/彩带动画效果（Confetti），恭喜你！
  • 一个大大的"Congratulations!"标题
  • 下面显示一个URL地址，类似：my-website.vercel.app

这个URL就是你网站的地址！点击它，你的网站就在新标签页中打开了——现在它是一个真正的、任何人都能访问的网站了！

把链接发给任何人！ 现在你可以把这个URL复制下来，发给你的家人、朋友。他们不管用手机还是电脑，打开这个链接就能看到你的网站。你真正做到了从0到上线！

第七步：以后怎么更新网站

网站上线之后，你肯定想改内容、加功能。怎么更新呢？超简单。

7.1 修改你的网站

  1. 用AI重新生成你想要的新代码（告诉AI"帮我修改xxx，增加yyy"）
  2. 把新的代码复制、保存为 index.html（覆盖旧的就行）

7.2 上传新版本到GitHub

  1. 打开 github.com/你的用户名/my-website
  2. 你会看到仓库里的 index.html 文件，点击它
  3. 在文件内容页面的右上角，有一个垃圾桶图标（删除按钮），点击它
  4. 页面会问你是否确认删除，输入提交信息 "删除旧版本"
  5. 点击绿色的"Commit changes"按钮
  6. 回到仓库主页，再次点击"Add file" → "Upload files"
  7. 把新的 index.html 拖进去
  8. 输入提交信息 "更新网站内容"
  9. 点击"Commit changes"

7.3 Vercel自动更新

最棒的地方在这里：Vercel会自动检测到GitHub的变化，然后自动重新部署！

你不需要再操作Vercel了。等1-2分钟，刷新你的网站，就能看到最新的内容了。

贴心提示： 如果你想更省事，可以不删除旧文件，直接上传新文件覆盖。但在GitHub网页上，你需要先删除再上传。如果你想避免每次删除的麻烦，可以考虑学习使用GitHub Desktop这个软件，它可以让更新文件变得更简单（不过不在本篇教程范围内）。

常见问题

Q: 页面显示404怎么办？

404表示文件没找到。最可能的原因是：你的index.html文件没有在仓库的根目录下，或者在保存时命名不对（比如保存成了index.html.txt）。检查一下你的GitHub仓库，确认index.html文件直接出现在仓库文件列表中，而不是在某个文件夹里面。如果不在根目录，删除后重新上传到根目录。

Q: Vercel部署报错怎么办？

部署报错最常见的原因是Framework Preset选错了。确保你在Configure Project页面中，Framework Preset选择的是"Other"，而不是"Next.js"或"React"等框架。如果还是报错，可以检查index.html文件内容是否完整，有没有被截断。

Q: 可以绑定自己的域名吗？

可以。在Vercel的项目页面中，点击顶部的"Settings"标签 → 左侧菜单选"Domains" → 输入你的域名（比如 yourname.com）→ 点击"Add"按钮 → 按照Vercel的提示在你的域名管理后台添加DNS解析记录。Vercel的免费套餐支持绑定自定义域名。

Q: 这个收费吗？

GitHub的免费套餐够你用。Vercel的免费套餐（Hobby计划）也完全够用，每月有100GB的流量和100小时的构建时长，对于个人网站来说绰绰有余。不需要绑定信用卡。

Q: 我的网站访问速度慢怎么办？

Vercel的服务器在国外，国内访问速度可能不太理想。如果你的目标用户主要在国内，可以考虑使用国内的云服务商，比如阿里云的OSS静态网站托管或腾讯云的静态网站托管。操作逻辑类似，但设置界面不同。

Q: 我想添加更多页面怎么办？

让AI帮你生成多个HTML文件，比如 about.html、contact.html、portfolio.html。然后把这些文件一起上传到GitHub仓库的根目录。Vercel会自动部署所有文件。在网页中通过链接（如 <a href="about.html">关于我</a>）来相互跳转。

Q: 我可以把图片放在网站上吗？

可以。把图片文件和HTML文件放在一起上传到GitHub仓库。在HTML中使用相对路径引用图片，比如 <img src="my-photo.jpg">。注意图片不要太大，建议压缩后再上传。

总结

你刚刚完成了人生中第一次从0到1的网站部署！用AI生成代码、用GitHub管理文件、用Vercel部署上线。整个过程不需要写一行代码，不需要懂任何编程知识。现在你有了一个属于自己的网站，任何人都可以通过链接访问它。

回顾一下你走过的路：

  • 和AI对话，让它帮你生成了一个网站
  • 注册了GitHub账号，学会了上传文件
  • 注册了Vercel，一键部署上线
  • 获得了一个可以分享的网站链接

这只是一个开始。你的网站可以不断完善：加更多页面、换更漂亮的风格、添加图片和动画效果——所有这些都可以通过和AI对话来完成。每次修改完，按照第七步的方法更新到GitHub就行了。

如果你感觉这篇教程对你有帮助，可以看看我们站上的其他文章，了解更多用AI做网站的技巧和思路。
