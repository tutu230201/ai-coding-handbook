📖 AI编程术语表

AI编程

利用人工智能技术辅助或自动完成代码编写、调试、重构等软件开发任务的过程。主流工具包括Claude Code、Cursor、GitHub Copilot等。

Prompt / 提示词

用户向AI模型输入的指令或问题。在编程场景中，好的Prompt能够显著提升AI生成代码的质量和准确性。

Token

AI模型处理文本的基本单位。在中文场景下，一个Token约等于1-2个汉字。模型的上下文窗口大小以Token数衡量。

上下文窗口 (Context Window)

AI模型一次能处理的最大Token数量。较大的上下文窗口允许模型理解更多的代码上下文。Claude 4.7支持200K Token的上下文。

幻觉 (Hallucination)

AI模型生成看似合理但实际错误的内容。在编程中表现为生成不存在的API、错误的逻辑或虚构的库函数。

RAG (检索增强生成)

通过检索外部知识库来增强AI生成内容的技术。在编程场景中，可以让AI基于项目代码库或文档来回答问题。

代码补全 (Code Completion)

AI根据当前代码上下文，实时预测并建议下一步要写的代码。GitHub Copilot的Tab补全是典型应用。

AI-First IDE

以AI为核心设计理念的集成开发环境，AI功能深度集成在编辑器的每个交互中。Cursor和Windsurf是代表产品。

Prompt工程 (Prompt Engineering)

系统性地设计和优化提示词的方法论。好的Prompt工程能显著提升AI输出的质量和可靠性。

多模态 (Multimodal)

AI模型能够理解和处理多种类型的数据（文本、图片、代码等）。Claude和GPT-4o均支持多模态输入。

Agent / 智能体

能够自主规划和执行任务的AI系统。在编程领域，AI Agent可以独立完成需求分析、代码编写、测试和部署等步骤。

Fine-tuning / 微调

在预训练模型基础上，使用特定领域的数据进一步训练，使模型在特定任务上表现更好。
