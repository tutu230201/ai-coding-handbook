用Cursor + Claude构建全栈博客

📅 2026-05-14 ⏱ 25分钟 中级
全栈开发 Cursor Claude

1. 项目概述：我们要构建什么

在这个教程中，我们将使用 Cursor 编辑器 配合 Claude AI，从零构建一个功能完整的全栈博客应用。整个开发过程将充分展示 AI 辅助编程如何大幅提升开发效率，让你体验从需求分析到代码生成的完整工作流。

我们的博客系统包含以下核心功能：

  • 文章列表展示页，支持分页浏览
  • 文章详情页，支持 Markdown 渲染展示
  • 后台管理界面，支持文章的创建、编辑和删除（CRUD）
  • 基于标签的文章分类系统
  • 响应式设计，适配桌面和移动设备

技术栈选择

技术用途选择理由
Next.js 14 (App Router)全栈框架前后端一体，AI 生成效率最高
Tailwind CSS样式框架类名即样式，AI 输出准确可控
Prisma + SQLite数据库 ORM零配置启动，后期可无缝迁移
Cursor + ClaudeAI 编程助手上下文理解强，代码生成质量高

选择这套技术栈的关键考量是：Next.js 的 App Router 模式将前后端统一在一个项目中，使得 AI 能够更好地理解全局上下文，生成的代码前后衔接更自然。而 SQLite 的零配置特性让我们可以跳过复杂的环境搭建，立刻进入编码阶段。

2. 环境准备

在开始之前，请确保你的开发环境已经准备就绪。以下是所需的基础工具：

  1. Node.js (v18.17 或更高版本) — 访问 nodejs.org 下载安装
  2. Cursor 编辑器 — 从 cursor.sh 下载并安装
  3. Git — 用于版本管理（可选但推荐）

安装完成后，打开终端验证环境：

node --version
# 输出示例: v18.17.1

npm --version
# 输出示例: 9.6.7

打开 Cursor，按 Cmd+K (Mac) 或 Ctrl+K (Windows/Linux) 打开 AI 对话面板。在右上角的模型选择器中确保选择 Claude 模型——这是获得最佳代码生成效果的关键。

3. 用AI生成项目骨架

这是本教程最激动人心的部分——我们将直接向 Claude 描述需求，让它来搭建整个项目的骨架。

3.1 初始化 Next.js 项目

首先在终端中执行以下命令创建 Next.js 项目：

npx create-next-app@latest ai-blog --typescript --tailwind --eslint
cd ai-blog

项目创建完成后，在 Cursor 中打开项目文件夹。现在，让我们向 Claude 提出第一个需求：

Claude 提示词示例：
"我想给这个 Next.js 博客项目添加 Prisma ORM 和 SQLite 数据库支持。请帮我完成以下工作：1) 安装 prisma 和 @prisma/client 依赖 2) 初始化 Prisma 并配置 SQLite 3) 创建一个 Post 数据模型，包含 id, title, slug, content, excerpt, tags, published, createdAt, updatedAt 字段 4) 生成 Prisma Client。请给出完整的终端命令和配置文件。"

Claude 会为你生成类似下面的 Prisma Schema：

// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

model Post {
  id        Int      @id @default(autoincrement())
  title     String
  slug      String   @unique
  content   String
  excerpt   String?
  tags      String   @default("")
  published Boolean  @default(false)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

3.2 创建数据库工具函数

接下来，让 Claude 帮你创建数据库连接客户端。在对话中继续输入：

"请帮我创建一个 lib/prisma.ts 文件，用于初始化 PrismaClient 单例，避免开发环境热重载时创建多个连接。同时创建 lib/posts.ts 工具函数文件，包含 getAllPosts、getPostBySlug、createPost、updatePost、deletePost 五个函数。"

Claude 会生成类似下面的代码：

// lib/prisma.ts
import { PrismaClient } from '@prisma/client'

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined
}

export const prisma = globalForPrisma.prisma ?? new PrismaClient()

if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma

export default prisma

5. 后端API开发

Next.js App Router 天然支持 API Routes，我们可以用 AI 快速生成完整的 RESTful API。以下是 Claude 为我们生成的文章创建 API：

// app/api/posts/route.ts
import { NextRequest, NextResponse } from 'next/server'
import prisma from '@/lib/prisma'

// GET /api/posts — 获取文章列表
export async function GET() {
  try {
    const posts = await prisma.post.findMany({
      orderBy: { createdAt: 'desc' }
    })
    return NextResponse.json(posts)
  } catch (error) {
    return NextResponse.json(
      { error: '获取文章列表失败' },
      { status: 500 }
    )
  }
}

// POST /api/posts — 创建新文章
export async function POST(request: NextRequest) {
  try {
    const body = await request.json()
    const { title, slug, content, excerpt, tags } = body

    // 验证必填字段
    if (!title || !slug || !content) {
      return NextResponse.json(
        { error: '标题、路径标识和内容为必填项' },
        { status: 400 }
      )
    }

    // 检查 slug 是否已存在
    const existing = await prisma.post.findUnique({ where: { slug } })
    if (existing) {
      return NextResponse.json(
        { error: '该路径标识已被使用' },
        { status: 409 }
      )
    }

    const post = await prisma.post.create({
      data: { title, slug, content, excerpt, tags: tags || '' }
    })
    return NextResponse.json(post, { status: 201 })
  } catch (error) {
    return NextResponse.json(
      { error: '创建文章失败' },
      { status: 500 }
    )
  }
}

AI 生成的 API 代码不仅逻辑完整，还包含了必要的错误处理和输入验证。你只需要在对话中补充 "请添加输入校验" 或 "请处理边界情况" 等要求，Claude 就会自动完善这些细节。

6. 数据库集成

完成 API 和页面开发后，我们需要初始化数据库并写入示例数据。以下步骤可以在 Claude 的指导下完成：

6.1 运行数据库迁移

7. 部署上线

项目开发完成后，部署到 Vercel 是最简便的选择——Vercel 本身就是 Next.js 的创建者，提供了最完善的部署支持。

7.1 构建配置

由于我们使用了 SQLite，需要在构建时做一些特殊配置。让 Claude 帮我们生成 vercel.json：

{
  "buildCommand": "npx prisma generate && next build",
  "outputDirectory": ".next",
  "installCommand": "npm install"
}

7.2 部署步骤

  1. 将代码推送到 GitHub 仓库
  2. 登录 Vercel，点击 "Add New Project"
  3. 导入你的 GitHub 仓库
  4. 在环境变量中添加 DATABASE_URL（SQLite 文件路径）
  5. 点击 "Deploy"，等待部署完成

需要注意的是，SQLite 在 Vercel 的无服务器环境中存在局限性——每次部署或冷启动时数据会重置。对于生产环境，建议后续迁移到 PostgreSQL（Vercel Postgres 或 Neon 都是不错的选择）。你可以直接告诉 Claude "请帮我将数据库从 SQLite 迁移到 PostgreSQL"，AI 会自动帮你完成 Schema 调整和迁移脚本的生成。

8. 总结

通过这个教程，我们完整地体验了 AI 辅助全栈开发 的工作流程。回顾整个过程，有几点值得强调：

  • 描述能力决定产出质量：向 AI 描述需求时越具体、越结构化，得到的代码就越接近预期。包含技术栈、样式要求、边界条件等细节的提示词能显著提升结果质量。
  • 迭代式开发：不要期望一次提示就能得到完美代码。先让 AI 生成基础版本，然后逐步提出修改要求，这种迭代方式远比手动修改高效。
  • 理解代码仍然重要：AI 可以大幅提升编码速度，但你仍然需要理解生成的代码，至少要知道每个文件的作用和它们之间的调用关系，这样才能在遇到问题时准确定位。
